# ADR-0002 — Go 1.25 toolchain

*Statut : Accepté · Daté 2026-05-24 (rétroactif M0) · Auteurs : thread principal*

## Contexte

TauGo est un projet greenfield sans base de code héritée. Le choix du langage et de la
version détermine l'outillage de test natif (fuzz, race detector), la compatibilité
de la chaîne CI, et l'alignement avec les projets de référence.

Trois candidats ont été évalués à M0 :

| Candidat | Justification évaluée |
|---|---|
| Go 1.21 LTS | Stabilité maximale, fuzz natif (1.18+), errgroup dispo. |
| Go 1.23 stable | Améliorations iterator range, log/slog stable. |
| Go 1.25+ (cible) | Alignement FibGo, toolchain auto-download, performances GC améliorées. |

PRD §13 prescrit : « Go 1.25+ avec toolchain 1.26.x — aligné FibGo ». Le calque FibGo
constitue la contrainte opérationnelle principale : FibGo (`agbruneau/FibGo`) utilise
Go 1.25 avec golangci-lint v1.64.8 épinglé, et TauGo emprunte son `.golangci.yml`,
son `arch_test.go` et sa structure `internal/calibration/`.

La divergence de version entre TauGo et FibGo engendrerait une dette de maintenance
immédiate sur les configurations partagées (linter, CI matrix).

## Décision

TauGo utilise **Go 1.25.0+** avec la directive `toolchain 1.26.x` dans `go.mod`.
La directive `toolchain` permet le téléchargement automatique du compilateur correct
sans épinglage manuel dans les images CI.

Points fixes de la décision :

- **golangci-lint v1.64.8 épinglé** — même version que FibGo, config calque direct.
  24 linters actifs (voir `.golangci.yml`).
- **stdlib autorisée** : `log/slog`, `math/big`, `sync/atomic`, `golang.org/x/sync/errgroup`.
- **`-race` via CGO** — activé en CI Linux/macOS uniquement. Windows sans gcc ne
  dispose pas du race detector ; la CI 3-OS compense.
- **Fuzz natif** (`go test -fuzz`) — disponible depuis Go 1.18, utilisé pour I1-I5.
  Pas de dépendance à un framework de fuzz tiers.
- **`t.Parallel()` systématique** — cible 100 % des tests unitaires ; performant
  depuis Go 1.7, stabilisé sur le scheduler M:N actuel.

## Conséquences

**Positives :**
- Alignement immédiat avec FibGo : `.golangci.yml`, `arch_test.go` et
  `internal/calibration/` peuvent être portés sans adaptation de version.
- Le fuzz natif couvre I1-I5 sans dépendance externe ; `-fuzztime=30s` en CI courte,
  `-fuzztime=24h` en CI nocturne.
- La directive `toolchain` dans `go.mod` rend la version reproductible pour tout
  contributeur sans installation manuelle.
- `errgroup` simplifie la gestion concurrente dans l'orchestrateur (PRD §10).

**Négatives :**
- Incompatibilité avec Go < 1.25 — acceptable pour un projet greenfield sans
  contrainte de rétrocompatibilité.
- Windows sans gcc : pas de `-race` local. Contournement : `make test-short` en
  développement Windows, race couvert par CI Linux.
- Go 1.25 n'est pas encore LTS au moment du choix initial (M0 2026). Risque faible :
  l'équipe Go garantit la compatibilité descendante par politique explicite.

**Neutres :**
- `math/rand/v2` (Go 1.22) disponible mais non requis dans le domaine τ.

## Alternatives rejetées

1. **Go 1.21 LTS** — version stable et supportée, mais divergerait de FibGo dès M0.
   Le portage des configurations golangci-lint aurait nécessité des adaptations
   (certains linters de v1.64.8 ciblent des constructs Go 1.22+). Rejet : coût de
   maintenance supérieur au gain de stabilité.

2. **Go 1.23 stable** — aucun gain substantiel par rapport à 1.25 pour TauGo.
   L'alignement FibGo prime sur la stabilité marginale de 1.23. Rejet : neutre sur le
   fond, négatif sur l'alignement.

3. **Rust** — typage fort, performances, fuzz mature (cargo-fuzz / libFuzzer).
   Rejeté pour deux raisons : (a) l'écosystème d'intégration agentique en Rust est
   moins mature qu'en Go en 2026 ; (b) l'équipe et les projets de référence
   (FibGo, AgentMeshKafka) sont en Go — diversifier le langage aurait fragmenté
   la base de connaissance.

4. **Python** — productivité initiale élevée, mais absence de typage statique
   fort et performances insuffisantes pour un kernel appelé à haute fréquence.
   Incompatible avec la contrainte de race-safety et de fuzz natif.

## Renvois

- PRD §13 (choix du langage et de la version)
- `go.mod` (directive `go 1.25`, `toolchain`)
- `.golangci.yml` (24 linters, version v1.64.8)
- FibGo `CLAUDE.md` (calque d'alignement)
- CLAUDE.md §Commandes essentielles (matrice CI 3 OS)

*Statut : Confirmé — aucune évolution de version prévue avant M6 release.*
